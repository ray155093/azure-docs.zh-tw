# <a name="how-to-use-the-azure-marketplace-seller-insights-portal"></a>如何使用 Azure Marketplace Seller Insights 入口網站

本文討論系統管理員如何存取 Seller Insights 入口網站、新增或刪除使用者，以及設定使用者的權限。

## <a name="onboard-users"></a>加入使用者

本節討論系統管理員如何第一次存取 Seller Insights 入口網站。 本節中的工作保留供系統管理員使用。 


### <a name="overview-of-roles"></a>角色概觀

Seller Insights 入口網站系統管理員可新增使用者、編輯使用者，以及刪除使用者。

Seller Insights 入口網站使用者可以檢視您組織可用的所有資料。

需要使用有效的 [Microsoft 帳戶](http://signup.live.com)或您組織的公司或學校帳戶，來驗證系統管理員和使用者。 請確定新的帳戶識別碼是與受監視的信箱相關聯。

![帳戶類型][5]



### <a name="invitation-to-activate-your-account"></a>啟用帳戶的邀請

登入 Seller Insights 入口網站之前，系統管理員應該識別他們想要設定為使用者之個人的帳戶。 設定具有公司或學校帳戶或者 Microsoft 帳戶的使用者。

為了啟用您的服務，系統管理員應該前往邀請電子郵件中所顯示的 [URL 位址](https://reports.azure.com)。

>[!NOTE]
>如果系統管理員電子郵件列為 Microsoft 帳戶，但您尚未建立與寄給您邀請的電子郵件地址建立關聯的 Microsoft 帳戶，則在登入之前必須[建立帳戶](https://signup.live.com)。


### <a name="administrator-sign-in-to-the-seller-insights-portal"></a>系統管理員的 Seller Insights 入口網站登入

1. 按一下傳送給您的電子郵件中的 URL，以登入 Seller Insights 入口網站。
2. 輸入系統管理員電子郵件地址和密碼。
3. 按一下 [登入] 按鈕。
4. 開始管理您的帳戶。 新增使用者、設定權限及編輯使用者。
5. 請確定您的使用者知道 Seller Insights 入口網站的 URL。

    ![登入頁面][6]

## <a name="user-sign-in-to-the-portal"></a>使用者的入口網站登入

系統管理員應該透過電子郵件通知您，表示已建立您的帳戶，並與您分享 Seller Insights 入口網站的 [URL 位址](https://reports.azure.com)。

>[!NOTE]
>系統管理員是將解決方案發佈到 Azure Marketplace 之帳戶的擁有者。 此帳戶的擁有者可以將存取權提供給其他人。  如果您能夠以共同管理員身分存取發佈入口網站，則可以依照下列步驟找到系統管理員地址︰
>1. 登入發佈入口網站。
>2. 選取 [發行者] 索引標籤。
>3. 開啟相關的發行者 (通常只有一個)。
>4. 按一下左下方的 [系統管理員]。
>5. 您要尋找的系統管理員電子郵件地址會標示為「擁有者」。


您可以使用個人 Microsoft 帳戶或者公司或學校帳戶來登入 Seller Insights 入口網站。 帳戶類型取決於系統管理員在您的設定檔中如何設定。 您的系統管理員應該通知您將會使用的帳戶。

![帳戶類型][1]



**歡迎使用 Seller Insights 入口網站！**

1. 登入之後，即會進入 [摘要] 頁面。
2. 請花幾分鐘的時間來閱讀＜隱私權原則＞。
3. 若有與 Seller Insights 入口網站相關的問題，您隨時可以透過電子郵件**與我們連絡**。
4. 您可以使用頁面頂端的索引標籤來瀏覽 Seller Insights 入口網站。  尋找本指南頁面頂端的索引標籤，協助您熟悉 Seller Insights 入口網站。
5. 隨時按一下微笑圖示，即可傳送與 Seller Insights 入口網站相關的意見反應。
6. 若要登出 Seller Insights 入口網站，請按一下您在右上角的名稱。

    ![預設登陸頁面][2]


## <a name="summary-view"></a>摘要檢視


1. 選取 [摘要] 索引標籤。
2. 在這裡輸入您想要之期間的 [開始日期]，然後按一下 [套用]。 會顯示三個月的資料。
3. 這裡顯示您選取的三個月期間的新訂單趨勢資料。
4. 這裡依通路和優惠類型顯示所選取期間的新訂單。
5. 從這個面板中選取檢視。 **前 5 個市場**、**前 5 個熱門優惠**或**前 5 名客戶**。
6. 這裡顯示所選取期間付費優惠的支付趨勢。 如果您只有 BYOL 或第 1 方優惠，則不會顯示此面板。

    ![摘要檢視][7]

## <a name="orders-and-usage"></a>訂單和使用量

在 [面板 D] 中，列出每個人的訂用帳戶資料。  您可以從這個面板將資料下載為 Excel 或 CSV 格式。

## <a name="user-management"></a>使用者管理

本節中的工作保留供系統管理員使用。 使用下列程序，從 Seller Insights 入口網站新增使用者、變更權限，或刪除使用者。


### <a name="add-user"></a>新增使用者

本節示範如何在 Seller Insights 入口網站中新增使用者或變更其權限。


1. 選取 [使用者管理] 索引標籤。
2. 若要新增使用者，請選取 [新增使用者]。
3. 輸入與其 Microsoft 帳戶或者其公司帳戶或學校帳戶建立關聯的電子郵件地址。 然後按一下 [繼續]。
4. 身為系統管理員，您可以指派唯讀報告存取權給使用者。
5. 輸入使用者的詳細資料，包括 [名字]、[姓氏] 和 [電子郵件])，然後按一下 [繼續]。
6. 選取您想要配置給使用者的權限，然後按一下 [繼續]。
7. 確定您所輸入的資訊正確，然後按一下 [儲存] 以繼續。  您已經成功新增使用者。

  ![新增使用者面板][29]


### <a name="edit-user"></a>編輯使用者

1. 選取 [使用者管理] 索引標籤，然後選取 [編輯使用者]。
2. 尋找您想要修改其權限的使用者。  您也可以新增使用者。 按一下 [加入使用者] 。
3. 按一下您要移除使用者所擁有的權限。  您也可以刪除使用者。 按一下 [刪除使用者]。
4. 按一下 [儲存] 以繼續。  您已成功編輯使用者的權限。

  ![編輯使用者面板][30]

## <a name="help-support-requests"></a>協助支援要求


### <a name="access-the-support-request-format"></a>存取支援要求格式

1. 選取 [說明] 索引標籤。
2. 選取 [支援和文件]，然後選取 [啟動要求]。
3. 您也可以從**與我們連絡**連結提交支援要求。 此連結會出現在應用程式中每個頁面的底部。
4. 將會開啟新的索引標籤，讓您提交支援要求。

  ![存取支援表單][31]

### <a name="review-helpful-information"></a>檢閱有用資訊

1. 若要提交與報告有關的支援要求，請從 [問題類型] 中選取 [報告和見解]。
2. 選取類別。  選項是 [與報告相關的問題] 和 [其他]。
3. 將會顯示一些有用的連結，或許可以解答您的疑問。  提交支援要求之前，請先查看這些連結。
4. 如果您仍然需要提交支援要求，請選取 [啟動要求]。

  ![編輯使用者面板][32]


### <a name="enter-information-and-submit-the-request"></a>輸入資訊，然後提出要求。

1. 輸入您的連絡資訊、時區、區域和語言喜好設定，然後選取 [繼續]。
2. 提供事件的標題，並盡可能提供最多資訊。  您也可以上傳支援檔案，例如螢幕擷取畫面，然後選取 [提交]。
3. 您會收到支援要求成功的確認。
4. 您隨時可以檢查支援要求的狀態。

  ![存取支援表單][33]


## <a name="reset-a-password"></a>重設密碼

**帳戶和密碼驗證**

1. 「驗證您使用的帳戶已註冊，但密碼無效。」 如果您輸入已在 Azure Active Directory 中註冊的帳戶，則會自動提示您選擇此帳戶的登入方式。 您可以使用公司或學校帳戶或者 Microsoft 帳戶作為您的個人帳戶。
2. 請前往 [https://reports.azure.com](https://reports.azure.com)，並登出您可能已登入的任何帳戶。 您也可以在私密視窗內開啟它，以確保沒有其他帳戶登入。
3. 輸入發行者電子郵件地址，但不要輸入密碼。 如果已在 Azure Active Directory 中註冊，則會重新導向至步驟 4。

    >[!NOTE]
    >系統管理員帳戶已註冊。 也會註冊由系統管理員帳戶所設定的公司或學校帳戶。

4. 選擇 [公司或學校帳戶]。 公司或學校帳戶是由已向 Azure 註冊的公司所設定。 

    >[!NOTE]
    >使用者可以使用其組織電子郵件地址來設定 Microsoft 帳戶。 在此案例中，組織後來加入 Azure，並將相同的電子郵件地址註冊為公司或學校帳戶。 此使用者的 Microsoft 帳戶密碼，可以與他們用來建立公司或學校帳戶的密碼相同。 此使用者的公司或學校帳戶密碼，也可以與其組織所使用的密碼不同。

5. 針對 Seller Insights 入口網站，當您選取 [公司或學校帳戶] 時，請確定您使用公司或學校帳戶密碼，而非 Microsoft 帳戶密碼。
6. 公司或學校帳戶會將您導向發行者頁面以進行登入。
7. 請在此頁面驗證公司或學校帳戶的密碼有效。 如果無效，請繼續閱讀本文的「密碼重設」一節。



**密碼重設**

請使用下列 Microsoft 以及公司或學校帳戶來重設密碼。

>[!NOTE]
>針對系統管理員帳戶所設定的公司或學校帳戶，如果系統管理員已開啟可讓您重設密碼的功能，您就能夠重設自己的密碼。 否則，密碼重設表單會您選擇連絡系統管理員。

* [Microsoft 帳戶](https://account.live.com/ResetPassword.aspx)
![密碼重設 Microsoft 帳戶][3]

* [公司或學校帳戶](https://passwordreset.microsoftonline.com/)
![密碼重設公司或學校帳戶][4]
