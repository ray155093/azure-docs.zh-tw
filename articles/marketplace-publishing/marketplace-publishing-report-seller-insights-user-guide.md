# <a name="how-to-use-the-microsoft-azure-marketplace-seller-insights-portal"></a>如何使用 Microsoft Azure Marketplace Seller Insights Portal
## <a name="onboarding"></a>登入

>[!NOTE]
>僅限系統管理員。


### <a name="overview-of-roles"></a>角色概觀

本節將示範如何第一次存取 Seller Insights 入口網站。


![帳戶類型][5]

使用 Seller Insights 入口網站系統管理員來新增使用者、編輯使用者，以及刪除使用者。

Seller Insights 入口網站使用者可以檢視您的組織可用的所有資料。

需要使用有效的 Microsoft 帳戶 (LiveID http://signup.live.com) 或您的公司帳戶來驗證系統管理員和使用者。 請確定新的帳戶識別碼是與受監視的信箱相關聯。

### <a name="invitation-to-activate-your-account"></a>啟用帳戶的邀請

在理想情況下，登入 Seller Insights 入口網站之前，系統管理員應該識別他們想要設定為使用者之個人的帳戶。 帳戶可以是 Microsoft 帳戶或公司帳戶。

為了啟用您的服務，系統管理員應該移至邀請電子郵件中所列的 URL (https://reports.azure.com)。

>[!NOTE]
>如果系統管理員電子郵件列為 Microsoft 帳戶，但您尚未建立與寄給您邀請的電子郵件地址相關聯的 Microsoft 帳戶，則您在登入之前必須建立帳戶。 若要建立的帳戶，請移至 https://signup.live.command。


### <a name="sign-in-to-the-seller-insights-portal"></a>登入 Seller Insights 登入網站

1. 按一下傳送給您的電子郵件中的 URL，以登入 Seller Insights 入口網站。
2. 輸入系統管理員電子郵件地址和密碼。
3. 按一下 [登入] 按鈕。
4. 開始管理您的帳戶。 新增使用者、設定權限及編輯使用者。
5. 請確定您的使用者知道 Seller Insights 入口網站的 URL。

![登入頁面][6]

## <a name="sign-in"></a>Sign in

系統管理員應該透過電子郵件通知您，表示已建立您的帳戶，並與您分享 Seller Insights 入口網站的 URL (https://reports.azure.com)。

>[!NOTE]
>系統管理員是實際將解決方案發佈到 Azure Marketplace 之帳戶的擁有者。  此帳戶的擁有者可以將存取權提供給其他人。  如果您能夠以共同管理員身分存取發佈入口網站，則可以依照下列步驟找到系統管理員的地址︰
>1. 登入發佈入口網站。
>2. 選取 [發行者] 索引標籤。
>3. 開啟相關的發行者 (通常只有一個)。
>4. 按一下左下方的 [系統管理員]。
>5. 您要尋找的系統管理員電子郵件地址會標示為「擁有者」。


您可以使用個人 Microsoft 帳戶或公司帳戶來登入 Seller Insights 入口網站。 帳戶類型取決於系統管理員在您的設定檔中如何設定。 您的系統管理員應該通知您將會使用的帳戶。

![帳戶類型][1]



**歡迎使用 Seller Insights Portal！**

1. 登入之後，您將會進入 [摘要] 頁面。
2. 請花幾分鐘的時間來閱讀隱私權原則。
3. 若有與 Seller Insights 入口網站相關的問題，您隨時可以透過電子郵件**與我們連絡**。
4. 您可以使用頁面頂端的索引標籤來瀏覽 Seller Insights 入口網站。  尋找本指南頁面頂端的索引標籤，可協助您熟悉 Seller Insights Portal。
5. 隨時按一下微笑圖示，即可傳送與 Seller Insights 入口網站相關的意見反應。
6. 隨時在右上角按一下您的名稱，即可登出 Seller Insights 入口網站。

![預設登陸頁面][2]


## <a name="summary-view"></a>摘要檢視

1. 選取 [摘要] 索引標籤。
2. 在這裡輸入您想要之期間的 [開始日期]，然後按一下 [套用]。 將會顯示 3 個月的資料。
3. 這裡顯示您選取的 3 個月期間的新訂單趨勢資料。
4. 這裡依通路和優惠類型顯示所選取期間的新訂單。
5. 從這個面板中選取檢視。 **前 5 個市場**、**前 5 個熱門優惠**或**前 5 名客戶**。
6. 這裡顯示所選取期間付費優惠的支付趨勢。 如果您只有 BYOL 或第 1 方優惠，則不會顯示此面板。

![摘要檢視][7]

## <a name="orders-and-usage"></a>訂單和使用量
### <a name="overview"></a>概觀

A. 在 [面板 A] 中，您可以選取 [訂單和使用量] 索引標籤。  在這裡，您可以從 [報告類型] 選取 [訂單] 或 [使用量]，然後使用滑桿選取想要的資料。

B. 在 [面板 B] 中，您可以選取每月資料。 依據您在 [報告類型] 中的選擇，選取 [訂單] 或 [使用量]。出現的資料會反映您所輸入的日期，以及您在 [面板 A] 中使用滑桿所套用的篩選條件。

C. 在 [面板 C] 中，您可以選取 3 個月的趨勢資料。 依據您在 [報告類型] 中的選擇，選取 [訂單] 或 [使用量]。出現的資料會反映您所輸入的日期，以及您在 [面板 A] 中使用滑桿所套用的篩選條件。

D. 在 [面板 D] 中，列出每個人的訂用帳戶資料。  您可以從這個面板將資料下載為 Excel 或 CSV 格式。

## <a name="user-management"></a>使用者管理

>[!NOTE]
>僅限系統管理員。


### <a name="add-user"></a>新增使用者

本節將示範如何在 Seller Insights 入口網站將新的使用者加以新增或變更其權限。


1. 選取 [使用者管理] 索引標籤。
2. 若要新增使用者，請選取 [新增使用者]。
3. 輸入與其 Microsoft 帳戶或公司帳戶相關聯的電子郵件地址，然後按一下 [繼續]。
4. 身為系統管理員，您可以指派唯讀報告存取權給使用者。
5. 輸入使用者的詳細資料，包括 [名字]、[姓氏] 和 [電子郵件])，然後按一下 [繼續]。
6. 選取您想要配置給使用者的權限，然後按一下 [繼續]。
7. 確定您所輸入的資訊正確，然後按一下 [儲存] 以繼續。  您已經成功新增使用者。

![新增使用者面板][29]


### <a name="edit-user"></a>編輯使用者

1. 選取 [使用者管理] 索引標籤，然後選取 [編輯使用者]。
2. 尋找您想要修改其權限的使用者。  您也可以新增使用者。 按一下 [加入使用者] 。
3. 按一下您要移除使用者所擁有的權限。  您也可以刪除使用者。 按一下 [刪除使用者]。
4. 按一下 [儲存] 以繼續。  您已成功編輯使用者的權限。

![編輯使用者面板][30]

4. 將會開啟新的索引標籤，讓您提交支援要求。

![存取支援表單][31]

2. 選取類別。  選項是 [與報告相關的問題] 和 [其他]。
3. 將會顯示一些有用的連結，或許可以解答您的疑問。  提交支援要求之前，請先查看這些連結。
4. 如果您仍然需要提交支援要求，請選取 [啟動要求]。

![編輯使用者面板][32]

1. 輸入您的連絡資訊、時區、區域和語言喜好設定，然後選取 [繼續]。
2. 提供事件的標題，並盡可能提供最多資訊。  您也可以上傳支援檔案，例如螢幕擷取畫面，然後選取 [提交]。
3. 您會收到確認訊息，指出已成功提交支援要求。
4. 您隨時可以檢查支援要求的狀態。

![存取支援表單][33]


## <a name="reset-a-password"></a>重設密碼

**帳戶和密碼驗證**

1. 「驗證您使用的帳戶已註冊，但密碼無效。」 如果您輸入已在 Azure Active Directory 中註冊的帳戶，則會自動提示您選擇此帳戶的登入方式。 您將會使用公司或學校 (組織帳戶) 帳戶，或個人 Windows LiveID 帳戶。
2. 請移至 https://reports.azure.com，登出您可能已登入的任何帳戶。 您也可以在私密視窗內開啟它，以確保沒有其他帳戶登入。
3. 輸入發行者電子郵件地址，但不要輸入密碼。 如果已在 Azure Active Directory 中註冊，將會重新導向至「步驟 4」。

  >[!NOTE]
  >系統管理員帳戶已註冊。 由系統管理員帳戶所設定的組織帳戶也已註冊。

4. 選擇 [公司或學校帳戶]。 工作或學校帳戶 (也稱為組織帳戶) 是由已向 Azure 註冊的公司所設定。 個人帳戶 (也稱為 LiveID 帳戶) 是向 Microsoft 註冊的個人帳戶。

  >[!NOTE]
  >使用者可以使用其公司電子郵件地址來設定 LiveID 帳戶。 在此案例中，公司後來加入 Azure，並將相同的電子郵件地址註冊為組織帳戶。 這表示此使用者可能有 LiveID 帳戶及他們用來建立 LiveID 帳戶的密碼。 使用者也可能有組織帳戶及公司所使用的另一個密碼。

5. 針對 Seller Insights 入口網站，當您選取 [公司或學校帳戶] 時，請確定您是使用組織帳戶密碼組合，而非 LiveID 密碼組合。
6. 組織帳戶會重新導向至發行者登入頁面。
7. 請在此頁面驗證組織帳戶的密碼有效。 如果無效，請繼續閱讀本文的「密碼重設」一節。



**密碼重設**
